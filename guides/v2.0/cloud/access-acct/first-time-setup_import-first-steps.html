导入现存的代码到项目

你可以从一个空模板创建一个Magento企业版(云支持版)的项目，或导入现有代码来创建。我们推荐使用一个空模板来开始，然后这之基础上导入现有的Magento代码

你不能以导入现存代码的方式创建试验项目。

导入代码的先决条件

开始之前, 请完成以下操作:

  • 添加现存Magento Commerce的代码到git仓库。我们推荐使用GitHub.
  • 设置你的 本地开发环境.

  • 收集必要的信息：

    • 目标环境的SSH访问链接
    • 数据库认证信息

SSH访问云环境

要传输数据库备份和文件到Magento企业版(云支持版),你必须知道SSH访问链接。你可以使用magento-cloud命令行工具找到SSH的访问链接

magento-cloud environment:ssh –pipe

你必须在你的存有Cloud SSH密钥的机器上输入所有Magento企业版(云支持版)的命令，更多信息请参考启用ssh密钥登录和SSH和sFTP.

数据库认证信息

你须要你的Magento企业版(云支持版)数据库名和认证信息，如些你便导入Magento Commerce的数据。你可以在$MAGENTO_CLOUD_RELATIONSHIPS环境变量中找到数据库名和认证信息。

要找到Magento企业版(云支持版)的数据库访问信息：

  1. 使用SSH登录到你的远程仓库。

     magento-cloud ssh -p <project ID> -e <environment ID>

  2. 列出所有数据库信息：

    echo $MAGENTO_CLOUD_RELATIONSHIPS | base64 -d | json_pp

<pre class="no-copy">database" : [

         "username" : "user",
         "query" : {

上例中，数据库名是main，端口是3306，主机名是database.internal，root用户名是user，并且此用户没有密码。

云非安全基础URL

在你导入 Magento Commerce数据库信息到Magento企业版(云支持版)之后，你必须修改基础URL，这样你就可以访问Magento管理面板和网店前台了。

使用magento-cloud命令行工具查看基础URL:

magento-cloud url

导入现存代码的工作流

导入现存代码的完整的工作流程包括以下的步骤：

  1. 如果你没有项目，从模板创建一个新的项目.新的项目有针对Magento企业版(云支持版)的文件和目录。
  2. 例用Git用自己的代码替换内容 of this project with your code using Git.
  3. 导入你的Magento数据库到Magento企业版(云支持版)工程。
  4. 导入静态文件到你的Magento企业版(云支持版)工程。
  5. 拷贝你的Magento Commerce的密钥到你的Magento企业版(云支持版)工程，这个密钥在数据迁移和访问时是必需的。
  6. 清除缓存并验证项目是否导入成功.

创建一个新的Magento企业版(云支持版)项目

  1. 访问你的账号，打开你从Magento Cloud(accounts@magento.cloud)收到的邮件并点击 立即访问你的项目 ，或你也可以登录到你的Magento账号.

  2. 点击 此项目暂时还没有代码 下一步到填写项目名称

  3. 填写项目名称。

  4. 点击 从模板创建一个空站点 然后点击 继续。我们推荐你以Magento模板作为你的初始项目选项开始，如果你有一个已经存在的Magento部署，你可以在之后导入现存代码。

  5. 当弹出提示时，在弹出的输入框输入你的Magento Commerce的Magento authentication keys。你之前在Magento市场创建了这些密钥。输入私钥和公钥然后点击 完成

    密钥会被添加到auth.json文件中，并且此文件所有分支和部署环境都要有。

  6. 你的项目部署要等几分钟，完成这前会显示 Pending 的状态，如下图所示：

  7. 项目部署完后，成功 会显示在你的项目名称旁边
