Magento框架

概述

Magento框架控制应用程序组件如何交互，包括请求流程，路由，索引，缓存和exception处理。它提供了为创建包含业务逻辑的模块减少努力的服务。贡献了使Magento代码更加模块化以及减少依赖的目标。

主要的PHP软件组件被组织成叫作库逻辑组，所有的模块都可以调用。多数框架代码放置于领域层，或围住表示层、服务层及领域层。框架不包含业务逻辑。 虽然Magento框架不包含资源模型，但它包含一个代码库来帮助实现资源模型。

不要混淆Magento框架和承载Magento的Zend应用框架

你不应该修改框架的文件，虽然如果你正在扩展Magento,你必须知道如何调用Magento的库，典型地，你创建的模块将从框架目录中定义的类和接口继承。

职责

Magento框架提供帮助你减少创建包含业务逻辑的模块努力的库。

框架对潜在所有的模块有用的操作进行响应，包括：

  • 处理HTTP协议

  • 数据库和文件系统的交互

  • 渲染内容

组织

这是Magento的文件结构：

vendor/
    ../magento
        ../framework
lib/
    ../internal
        ../LinLibertineFont
    ../web

  • /vendor/magento/framework目录仅包含PHP代码。它们是代码库加上路由请求到模块(依次调用框架库)的应用程序入口点。例如，框架库帮助实现一个资源模型(基类和接口从这里继承)，但不是资源模型本身。某些库也支持CSS渲染。

  • /lib/internal 包含一些非PHP及PHP组件。非PHP框架库包含JavaScript和LESS/CSS

  • /lib/web 包含JavaScript和CSS/LESS文件。这些文件存在在web而不是internal，因为它们是web浏览器可访问的。而internal下的PHP代码则可以。(任何浏览器需要访问的代码都应放在web下，而其它的放到internal下)

vendor/magento/framework目录映射Magento\Framework 命名空间.

Magento框架的亮点

Magento框架(lib/internal/Magento/Framework/)提供了一系列稳健的功能。如果你是一个扩展开发者，你可能对这个框架的命名空间子集感兴趣。

命名空间 目的
Magento\Framework\DataObject 提供存储和通过魔术方法接收数据的标准功能。
Magento\Framework\Model 包含几乎所有Magento模型类要继承的基础模型类
Magento\Framework\Model\AbstractModel
Magento\Framework\Model\ResourceModel\AbstractResource
Magento\Framework\Controller 包含帮助返回不同类型结果集的所有类（例如，JSON或直接返回）
Magento\Framework\View 包含渲染页面和布局的代码
Magento\Framework\Data 包含处理表单的额外类
Magento\Framework\Url 包含查找Magento其它页的代码

其它在Magento\Framework下的命名空间将引起扩展开发者的兴趣：

命名空间 目的
Magento\Framework\ObjectManager 被用于提供依赖注入
Magento\Framework\App 包含知道Magento应用程序信息的框架代码。此代码引导应用程序且读取初始化配置。它也包含命令行工具、网页应用程序、定时任务的入口点。最终，它提供布署上下文环境(诸如读取数据库、语言、缓存系统的配置)路由请求。
Magento\Framework\Api 包含通过系统可扩展的对象的高级功能的基类(即，对象可以通过Magento市场的扩展扩展添加新数据的)
Magento\Framework\Config 包含一般配置读取器，每个配置文件有自己特定的从这些类继承的读取器。
Magento\Framework\Filesystem 包含处理读取和写入文件系统的类
Magento\Framework\HTTP\PhpEnvironment
Magento\Framework\Session
Magento\Framework\Stdlib\Cookie 处理HTTP请求/响应以及SESSION/COOKIE的代码
Magento\Framework\Exception Contains the basic exceptions that are thrown throughout the Magento codebase.
Magento\Framework\Event Contains the code that publishes synchronous events and that handles observers for any Magento event is handled here.
Magento\Framework\Validator 包含验证数据(货币校验，非空校验)的代码及处理任何Magento的事件观察者的代码。
